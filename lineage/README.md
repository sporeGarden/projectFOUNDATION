# lineage/

The unified scientific lineage that gives foundation its substance.

## Contents

- **THE_UNIFIED_LINEAGE.md** — The master document. Maps all 10 domain
  threads with their baseCamp papers, spring validations, contacts, and
  cross-thread connections. Start here to understand the foundation as
  a whole.

- **THREAD_INDEX.toml** — Machine-readable inventory of all 10 threads.
  Intended for tooling that needs to enumerate threads, look up springs
  per domain, or generate validation workload manifests.

## How to Read

Start with `THE_UNIFIED_LINEAGE.md` for the narrative arc: why the
lineage is unified, what each thread contains, and how threads
interconnect. Then use `THREAD_INDEX.toml` for structured lookups.

For detailed per-thread analysis, see `expressions/` — each expression
document zooms into one thread with paper-by-paper composition blueprints,
data target inventories, and petalTongue visualization maps.

## Historical Context

The whitePaper repository (`ecoPrimals/infra/whitePaper/gen4/foundations/`)
contains the personal and historical genesis of these ideas. This repo
is the living expression — the documents here are repo-native, updated
as the ecosystem evolves, and designed for operational use by
projectNUCLEUS validation runs and product teams.
